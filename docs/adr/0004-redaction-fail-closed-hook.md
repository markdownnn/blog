# ADR 0004 — 익명화 안전망: fail-closed pre-push 훅 (CI 대신)

- 상태: 채택
- 관련: [ADR 0001](0001-single-public-repo.md), [ADR 0003](0003-drafts-stay-local.md)

## 맥락

익명화 사고(회사명·인물·수치 노출)는 편도다 — 공개 사이트에 한 번 올라가 인덱싱되면 글을 지워도 검색에 남는다. repo가 단일 public이 되면서(ADR 0001) 실수의 대가가 더 커졌다.

v0.2는 CI 검사를 일부러 뺐고, 안전을 전적으로 Claude 스킬의 "기억"에 맡겼다. 문제: 컨셉이 "`git push` = 배포 끝"이라, 사용자가 스킬을 우회해 직접 `git push`하면 최종 스캔이 통째로 건너뛰어진다. AI의 기억은 비결정적이라 안전장치로 부적합하다.

## 결정

**기계식 pre-push 훅으로 "아는 금칙어의 실수 커밋"을 막는다.** (완벽한 방어가 아니라 덫.)

- `scripts/pre-push.sh`가 push되는 커밋 내용을 `~/.blog-redact-terms.txt`의 금칙어와 `grep` 대조. 걸리면 push 차단 + 라인 출력.
- **fail-closed**: 금칙어 파일이 **없거나 / 비었거나 / 너무 짧으면** 차단. (파일만 있고 내용이 없어 조용히 초록불 켜지는 구멍까지 막음.)

## 한계 (정직하게)

훅은 "최후 방어선"이 아니다. 다음으로 뚫린다:
- `git push --no-verify`로 훅 자체를 끌 수 있음.
- 외부 PR·GitHub 웹 편집은 로컬 훅을 거치지 않음(public repo).
- 훅 안 깐 다른 기기의 push도 안 거침.
- 수동 금칙어 목록에 없는 신원 정보는 못 잡음.

→ 훅은 *실수*를 잡는 덫이다. **진짜 방어는 "글 쓸 때 처음부터 익명화"(blog-draft 스킬) + 발행 전 사람 눈 검토.** 훅은 그 보완.

## 결과

- **좋아짐**: 아는 금칙어의 실수 유출을 결정적으로 차단. 안전 목록 무력화 시 조용하지 않게(fail-closed).
- **대가**: 목록 최신 유지 필요. 파일 없거나 빈약하면 push 막혀 살짝 귀찮음(의도된 것).
- **되돌리기**: 훅 제거는 쉬움. 그 순간 실수 그물이 사라짐.

## 대안

- **CI 기반 검사**: 서버에서 도는 결정적 검사라 확실하나, 로컬 금칙어 파일(`~/.blog-redact-terms.txt`, 어디에도 커밋 안 함)을 CI가 볼 수 없음 → 민감 목록을 CI 시크릿에 올려야 하는데 이는 유출면을 넓힘. 로컬 훅이 이 파일과 궁합이 맞아 채택.
- **스킬 기억에만 의존 (v0.2)**: 비결정적 + push 우회 가능. 기각.
